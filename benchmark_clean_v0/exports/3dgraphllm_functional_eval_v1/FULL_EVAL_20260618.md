## Interpretation

The reproduced original 3DGraphLLM pipeline can run the full FunGraph functional
500 split with the real SceneFun3D adapter packet. However, it almost never
emits a usable object decision: 474/500 predictions contain no object token, and
all 26 token-bearing predictions contain multiple valid object tokens rather
than a single localized answer. Only 1/500 predictions has the correct first
valid object token.

This is evidence of an answer-protocol and task mismatch for FunGraph functional
localization, not evidence that SceneFun3D adapter features solve the task. The
next adaptation step should make object selection explicit and evaluate whether
a controlled answer format improves object-token localization before claiming
multimodal functional reasoning.
